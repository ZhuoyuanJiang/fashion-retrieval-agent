# Plan 3 — Server baseline run + report

## Context

Plan 2 built the full baseline pipeline locally using a mock VLM and an oracle
captioner, with the real `SpeechQwen2VLCaptioner` backend deliberately stubbed
behind a `>= 14 GB VRAM` gate. All three milestone tests (M1–M3) pass locally.
The repo is pushed to `main`.

This plan picks up at the server step: activate real GPU inference, run the
pipeline on the FACap dress evaluation slice, and produce the
`Baseline_1_Report_20260430.md` that feeds into the next planning round.

**Key decisions carried in:**
- Default VLM: `speechqwen2vl` (Stage-1 model in text-only mode, per project
  narrative continuity). `qwen2vl` oracle/mock ablations are available as
  overrides if the default has issues.
- Eval slice: last 1000 triplets of FACap dress train set.
- Gallery: full 59 082 FACap dress targets.
- Text encoder: `all-MiniLM-L6-v2` (384-d, CPU-only).
- Model weights: `DanJZY/Qwen2-VL-7B-Speech` (base) +
  `DanJZY/Qwen2-VL-7B-Speech-LoRA` (Stage-2 LoRA).

## Milestones

### M1 — Environment + data setup

**Goal:** `fashion_retrieval` conda env created, annotation repos cloned,
symlinks in place so no large files land in `~`.

Steps:
1. Create scratch dirs on SSD:
   ```bash
   mkdir -p /ssd1/zhuoyuan/hf_cache
   mkdir -p /ssd1/zhuoyuan/facap-images
   ```
2. Symlink image cache into repo:
   ```bash
   ln -s /ssd1/zhuoyuan/facap-images \
         data_exploration/datasets/facap-images
   ```
3. Clone annotation repos (~240 MB text, stays on NAS):
   ```bash
   bash scripts/setup_datasets.sh
   ```
4. Create conda env (uses existing sibling `speechQwen2VL`):
   ```bash
   export SPEECHQWEN2VL_DIR=/home/zhuoyuan/projects/speechQwen2VL
   export HF_HOME=/ssd1/zhuoyuan/hf_cache
   bash scripts/setup_server.sh
   ```

**Done when:** `python -c "import transformers; print(transformers.__version__)"`
prints `4.56.0.dev0` inside the `fashion_retrieval` env.

### M2 — Smoke test (N_EVAL=50)

**Goal:** Verify the full pipeline (image prefetch → VLM caption → embed →
retrieve → metrics) runs to completion without errors on 50 queries. Downloads
Qwen2-VL-7B on first run (~15 GB to `/ssd1/zhuoyuan/hf_cache`).

```bash
conda activate fashion_retrieval
export HF_HOME=/ssd1/zhuoyuan/hf_cache
N_EVAL=50 RUN_NAME=smoke_real bash scripts/run_baseline_v1.sh
```

**Done when:** `runs/smoke_real/metrics.json` exists and contains valid
`recall@1`, `recall@5`, `recall@10`, `recall@50`, `median_rank`, `mean_rank`
fields. Exact numbers don't matter at this stage.

### M3 — Full baseline run (N_EVAL=1000)

**Goal:** Produce the headline baseline number on the full 1000-query
evaluation slice against the 59 082-target gallery.

```bash
export HF_HOME=/ssd1/zhuoyuan/hf_cache
bash scripts/run_baseline_v1.sh
```

Defaults: `N_EVAL=1000`, `DB_SIZE=59082`, `VLM=speechqwen2vl`,
`RUN_NAME=baseline_v1_speechqwen2vl`.

**Done when:** `runs/baseline_v1_speechqwen2vl/metrics.json` and
`runs/baseline_v1_speechqwen2vl/qualitative/results.jsonl` both exist.

### M4 — Baseline report

**Goal:** Write `Documentation/Baseline_1_Report_20260430.md` summarizing the
run results. This document feeds the next planning round (Plan_4 scope depends
on whether the baseline is strong or weak).

Required contents (from `docs/server_handoff.md`):
1. **Metrics table** — R@1 / R@5 / R@10 / R@50, median rank, mean rank.
2. **5 qualitative success cases** — reference image path, modification text,
   retrieved target image path, rank. All visually correct.
3. **5 qualitative failure cases** — same fields plus `failure_category`
   filled from the Plan_2 rubric:
   - `caption_wrong` — VLM caption misses the modification
   - `embedding_mismatch` — caption fine but SBERT places it far from target
   - `dataset_ambiguity` — multiple plausible targets exist
   - `visual_nuance_lost` — modification too subtle for text to capture
4. **Provenance** — `caption_db/config.json` content + exact command used.
5. **Interpretation paragraph** — is this baseline strong or weak?
   Strong → contrastive learning needs a more compelling angle; weak →
   contrastive learning has clear motivation.

**Done when:** `Documentation/Baseline_1_Report_20260430.md` is committed to
`main` with the above five sections filled in.

## Failure modes and mitigations

| Error | Cause | Fix |
|---|---|---|
| `RuntimeError: server-only: ... needs >= 14.0 GB VRAM` | Wrong GPU selected | Set `CUDA_VISIBLE_DEVICES=<id>` to a free RTX 3090 |
| `transformers.__version__` = `5.x` | Forks didn't override | Re-run `bash /home/zhuoyuan/projects/speechQwen2VL/scripts/setup_forks.sh` inside the activated env |
| HF 429 rate-limit during model download | No token set | `export HF_TOKEN=<token>` or `huggingface-cli login` |
| Stale-DB error | DB built with different args | Delete `runs/<run_name>/caption_db/` or use a fresh `--run-name` |
| NAS quota exceeded | Large file landed in `~` | Check `du -sh ~`; verify `HF_HOME` is `/ssd1/...` |
