# Plan ‚Äî Dataset Exploration for Audio-Conditioned Fashion CIR

## Context

**Goal for today:** verify whether the two candidate datasets (FACap, FashionIQ) actually
fit the "audio-conditioned composed fashion retrieval" task, and produce a concrete visual
sample to bring to the mentor meeting **tonight (2026-04-20)**.

**Project task (already settled in `Documentation/project_proposal_20260420.md` Part 4b):**
given a reference garment image + a spoken modification, retrieve the target garment.
Two methods are in scope ‚Äî (1) target-oriented caption generation + text retrieval (main),
(2) direct contrastive query/target embedding (extension). Both rely on
`<reference image, modification text, target image>` triplets, which we convert to
audio via TTS.

**Why this step matters:** the mentor asked for the proposal to be grounded in real data.
Before we refine method details or write training code, we need to confirm:

- The data format is truly composed retrieval (ref + modification ‚Üí target).
- The modifications are **fine-grained** (sleeve, length, formality, cut, pattern, fabric).
- Modifications can be TTS'd into natural-sounding speech.
- Target images either come with captions or can be captioned for the main-method baseline.
- Candidate pool / evaluation protocol is usable.

## Findings from background research (already done in this planning session)

### FashionIQ ‚Äî ready to use today

- Official repo: `https://github.com/XiaoxiaoGuo/fashion-iq` ‚Äî contains `captions/`,
  `image_splits/`, and a full `start_kit/` with `data_loader.py` and `eval.py`.
- 3 categories: `dress`, `shirt`, `toptee`.
- Triplet JSON schema (confirmed):
  ```json
  {"target": "B008BHCT58", "candidate": "B003FGW7MK",
   "captions": ["is solid black with no sleeves", "is black with straps"]}
  ```
  **Two short relative modifications per triplet**, naturally TTS-able.
- Split sizes: ~6k train / ~2k val / ~2k test per category.
- Images not in the repo ‚Äî Amazon URLs via `https://github.com/hongwang600/fashion-iq-metadata`
  (`asin2url.{dress,shirt,toptee}.txt`). Research spot-checked 3 URLs in 2026 ‚Üí HTTP 200.
- License: CDLA-Permissive (no blockers).
- Standard metric: Recall@10 + Recall@50 averaged across categories.

### FACap ‚Äî annotations likely accessible, training code "coming soon"

- Paper: `https://arxiv.org/abs/2507.07135` ‚Äî 227,680 triplets, 23-token average modification,
  82-token average target caption, built from Fashion200k + DeepFashion-MultiModal.
  Modifications auto-generated via VLM+GPT-4o-mini, fine-grained by construction.
- Project page: `https://fgxaos.github.io/facap-paper-website/`.
- Repo: `https://github.com/fgxaos/facap-sigir25-gennext` ‚Äî contains `data/`, `src/`,
  `requirements.txt`. README says **"The code will be released soon"** but provides
  `python src/run/download_datasets.py` with flags for FashionIQ and FACap paths.
- License: not stated on project page or README.
- Open question: whether the FACap-specific annotations (triplet JSON, modifications,
  target captions) are committed under `data/`, or whether the download script pulls
  from a remote that isn't public yet. **Must verify by actually cloning.**
