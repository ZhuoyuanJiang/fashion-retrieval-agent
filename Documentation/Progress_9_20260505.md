## 1. What was implemented

### 1.1 Code changes

- **`src/baseline/vlm_caption.py`** — added `PROMPT_TEMPLATE_DETAILED`
  alongside the original `PROMPT_TEMPLATE_CONCISE` (renamed for

  prompt_variant=...)`, stamped into `metrics.json["context"]
  ["prompt_variant"]` so any downstream analysis can tell which prompt
  produced a result.
- **`scripts/run_baseline_v1.sh`** — added `PROMPT_VARIANT` env var
  passthrough (default `concise`), printed in the run config banner,
  and forwarded to `--prompt-variant`.
- **`scripts/run_encoder_swap.sh`** — added `RUN_PREFIX` env var
  (defaults to `basename "$SOURCE_RUN"`) so the per-encoder run names
  follow the source-run name. Without this the script hardcoded
  `baseline_v1_speechqwen2vl_<slug>` and the v2 swap was effectively
  a no-op (all 11 encoders skipped because their v1 outputs already

- **`src/baseline/aggregate_encoder_results.py`** — added two CLI
  flags: `--source-prefix` (was hardcoded to
  `baseline_v1_speechqwen2vl`) and `--compare-prefix` (new). When
  `--compare-prefix` is set, emits a side-by-side R@K table with
  `Δ R@K` columns. The single-source mode is unchanged.

### 1.2 The new prompt

```python
PROMPT_TEMPLATE_DETAILED = (
    "Given the reference fashion image and the modification instruction, "
    "write a detailed caption describing the target fashion item after "
    "applying the modification. For example, you could consider aspects "
    "like silhouette, neckline, sleeves, fabric, color, pattern, or "
    "distinctive details."
)
```

Two changes from v1: `concise` → `detailed`, plus a deliberately soft
"For example, you could consider..." inspiration hint (not a
checklist). See Plan 9 §4 for the design rationale (why no example
vocabulary, why no bulleted format, why no `if visible` framing).

### 5.1 Findings

1. **New SOTA: `qwen3-emb-8b` on detailed prompt → R@1 = 0.290.**
   Beats the previous champion (marqo-fashionclip on concise prompt,
   R@1 = 0.258) by Δ = +0.032. R@10 = 0.586 (previous best 0.533) and
   R@50 = 0.710 (previous best 0.704). **The headline encoder choice
   for Phase A flips depending on prompt regime.** The
   "best encoder" recommendation now needs to be paired with
   "and which prompt".

2. **The Qwen3-Embedding family (0.6B / 4B / 8B) is the unambiguous
   winner under the detailed prompt.** All three sizes improve, with
   monotone scaling: 8B > 4B > 0.6B on Δ R@1. Their 8192-token context
   windows mean they see all of the longer captions in full, and they
   appear to be the encoders most able to extract useful signal from
   the additional length even with noise.

3. **Plan 9 §3's CLIP-token-cap prediction was partially right.**
   Marqo FashionSigLIP (64-token cap, smallest window) had the largest
   relative R@10 loss in the CLIP family (−0.035 vs Marqo
   FashionCLIP's −0.049 — though FashionCLIP also lost more on R@1
   because it had higher v1 baseline). The token-cap interaction is
   directional but isn't the whole story.

4. **BERT-retriever family (bge-large, e5-large-v2, mpnet-base) all
   regressed despite having 512-token windows that can fit the new
   captions.** Hallucinated attributes outweigh the extra context for
   them — the noise floor rises faster than the discriminative
   ceiling.

5. **MiniLM-L6 is the worst loser (−0.075 R@10, −0.091 R@50).**
   Confirms that the regression seen in §3 isn't an artifact of
   token-cap truncation (256 tokens still fits 199-char captions);
   it's that MiniLM's smaller capacity makes it most vulnerable to
   the noise/signal ratio dropping.

6. **CLIP-ViT-L-14 (general, not fashion-tuned) actually improved
   slightly on R@10/R@50** despite having the same 77-token cap as
   Marqo FashionCLIP. Tiny absolute numbers (R@10 0.083 → 0.097) but
   directionally counter-intuitive — possibly because the longer
   captions average over the truncation point more gracefully when
   the encoder has weaker domain-specific priors. Not worth chasing
   further; CLIP general is far below the practical frontier either
   way.

## 6. Mechanism analysis: why most encoders regressed

The result is counter-intuitive: longer captions share **more** vocabulary
with FACap target captions on average (mean token_overlap 0.21 → 0.317,
+50%), yet retrieval is **worse** for 8 of 11 encoders. This section
traces the mechanism through four pieces of evidence and three causal
claims.

### 6.1 The four-number evidence chain

| Metric | v1 | v2 | Reading |
|---|---|---|---|
| Caption length (median chars) | 92 | 199 | +116% — prompt change worked |
| Mean token_overlap (gen ∩ target / target) | ~0.21 | 0.317 | +50% — more vocabulary match |
| Spearman ρ (rank vs token_overlap) | **−0.306** (p=4e−23) | **−0.076** (p=0.016) | Correlation **collapsed** |
| Spearman ρ (rank vs len_ratio) | −0.216 | **+0.175** | Sign **flipped** |

Row 3 is the smoking gun. Token overlap went **up** in absolute terms but
**stopped predicting rank**. The only explanation is that the new
"matching" tokens are **generic high-frequency words** — they overlap with
many gallery dresses equally, contributing zero discriminative signal.

Row 4 is the reinforcement. Under v1, longer captions correlated with
*better* retrieval (more useful info). Under v2, longer captions correlate
with *worse* retrieval — the additional length is actively hurting, not
helping.

### 6.2 Three mechanisms acting together

**M1 — Hallucination dilution (the dominant effect).** The detailed
prompt asks the VLM to elaborate, but with no ground-truth anchor it
fills in attributes from generic priors. Smoke Q3 was the canonical
example: the modification said "round neckline / flared mid-calf
silhouette / gold-tone details / suede texture", but the VLM ignored
all of that and produced "deep V-neckline / silk or satin material /
straight simple silhouette" — completely different attributes. These
hallucinated tokens are valid English fashion vocabulary, so they
"match" many gallery dresses (boosting raw token_overlap), but they
pull the embedding direction away from the actual target.

**M2 — Modification-as-target overwrite.** FACap's modification text
is itself a detailed description of the target (e.g. *"The dress is
longer and made of linen, featuring a V-neckline and long sleeves..."*).
Under v1's concise prompt the VLM essentially paraphrases the
modification — faithful information preservation. Under v2's detailed
prompt the VLM tries to "add value" and ends up overriding the
modification's specific attributes with its own generic guesses. The
smoke Q1 case (caption byte-identical to modification) is actually
the *best* v2 outcome — the VLM just didn't elaborate.

**M3 — Embedding-direction shift.** Cosine similarity is direction-based.
Adding ~100 hallucinated tokens to a 92-char caption shifts the
normalized embedding toward the **mean dress caption** (because
hallucinated tokens are high-frequency vocabulary). This moves the
embedding *closer* to the average gallery item but *farther* from the
specific target. Hence rank degradation despite higher token overlap.

### 6.3 Why Qwen3-Embedding family escapes the trap

Qwen3-emb (0.6B / 4B / 8B) all improved monotonically with size
(Δ R@1 = +0.054 / +0.080 / +0.116). Three plausible reasons:

- **Long-context pretraining (8192-token windows).** These models saw
  long noisy text during pretraining; they learned to extract
  signal from filler. Smaller bidirectional encoders (BERT family) did
  not — they were trained on shorter, denser inputs.
- **LLM-decoder capacity.** 4–8B parameters give enough representational
  room to distinguish "noise word" from "signal word" in the input;
  MiniLM (22M) and BGE (335M) lack that capacity.
- **Modern instruction-following bias.** Qwen3 family was trained with
  retrieval-task instruction prefixes; even without our explicit
  prompt-engineering, it internally weighs "salient garment-attribute
  tokens" higher than "filler tokens".

This isn't proof — it's the most plausible account given the
size-monotone improvement and the architectural priors. A more rigorous
test would compare attention/contribution weights across token positions
in v1 vs v2, but that's out of scope here.

### 6.4 Why MiniLM-L6 lost worst (−0.075 R@10)

256-token window comfortably fits 199-char captions, so token-cap
truncation is *not* the cause. Three contributing factors instead:
- **Smallest capacity (22 M params)** → least room to filter noise.
- **Weakest baseline** → highest relative impact from any input quality
  shift; floor-effect plus headroom-loss combine.
- **No retrieval-prefix training** → less discrimination between salient
  vs. generic fashion vocabulary in the embedding.

### 6.5 What we cannot conclude

The above is a mechanism story consistent with the numbers, not a proof.
To rigorously isolate hallucination as the dominant cause we would need
to **hand-label** a sample of v2 captions for attribute-faithfulness
against the target image. Plan 4's `annotations.jsonl` scaffold is
exactly the right tool — random-sample 100 v2 queries, label each
caption with attribute-hallucination rate, then correlate hallucination
rate with rank. If hallucination rate ≥ 30% and Spearman rho with rank
is strongly positive, the mechanism is confirmed. Out of scope for this
progress entry; queued as a follow-up if we ever need to publish
attribution claims.

What the current evidence *does* support:
- The detailed prompt should NOT be the default for small encoders
  (MiniLM, BGE, mpnet, CLIP family).
- It IS the right default specifically for Qwen3-Embedding family.
- The Phase A best-encoder recommendation is now prompt-conditional
  (see §7).

## 7. Decision (per Plan 9 §7)

The result is the **mixed scenario**: "detailed > concise for
LLM-decoder family, ≤ for BERT/CLIP family." Plan 9's pre-written
guidance: "The 'best' encoder choice now depends on prompt — record
both as Phase A's final story."

Concrete next-step implications:

- **`qwen3-emb-8b` on detailed prompt is the new strong-baseline
  reference for Phase A.** The Phase A → Phase B handoff narrative
  should cite this number, not Marqo FashionCLIP's concise R@1 = 0.258.
- **Plan 5/6 contrastive should still be evaluated against
  `marqo-fashionclip` as its frozen image-tower** because that
  pipeline never goes through the caption step (it's image-to-image
  via contrastive embedding, captions irrelevant). The detailed-prompt
  result doesn't change Plan 5/6's reference.
- **Open question: would a different VLM (vanilla Qwen2-VL or GPT-4V)
  benefit *more* from the detailed prompt than speechQwen2VL does?**
  speechQwen2VL has a Stage-1 audio-tuning bias that may be hurting
  its long-form text generation more than vanilla Qwen2-VL would. This
  is a candidate for a separate Plan ("VLM swap with detailed prompt")
  but not done here.
- **No prompt Tier 2/3 escalation is needed.** The Tier 1 detailed
  prompt already produced materially longer captions (199 vs 92 char
  median); the issue isn't length, it's hallucination. Adding more
  structure to the prompt (Tier 2's "be specific" anchor, Tier 3's
  bulleted axes) is more likely to amplify hallucination than fix it.
