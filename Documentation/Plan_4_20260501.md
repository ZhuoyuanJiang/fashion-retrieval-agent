# Plan 4 — Caption generation analysis notebook (2026-05-01)

## Context

Phase A baseline + 11-encoder ablation are complete. The next decision
point is whether the remaining R@1 ceiling is bottlenecked by **caption
quality** (the speechQwen2VL VLM produces poor descriptions of the
target garment) vs. **retrieval** (the encoder fails to map a good
caption to the right gallery row). Before designing Plan_4.5 (VLM
swap) or Plan_5 (contrastive end-to-end), we need to actually *look at*
the generated captions and characterize their failure modes.

Goal: a dedicated, mentor-shareable notebook that loads the saved
captions from `runs/baseline_v1_speechqwen2vl/qualitative/results.jsonl`
and renders them next to the reference image, modification text,
target image, ground-truth target caption, and retrieval rank — with
both a per-query gallery and rank-stratified buckets, plus simple
caption-vs-target programmatic metrics. No baseline rerun required;
all 1000 captions are already saved.

## Notebook structure

Cells in order (jupytext percent format):

1. **Markdown header** — title, scope, source run path, what this
   notebook does NOT do (no rerun, no encoder swap).
2. **Imports + paths** — `REPO_ROOT`, `RUN_DIR =
   runs/baseline_v1_speechqwen2vl/`, `RESULTS = RUN_DIR /
   "qualitative/results.jsonl"`. Single config cell at the top so a
   future run (e.g. `runs/baseline_v1_qwen2vl/`) can be analyzed by
   changing one variable.
3. **Load data** — read `results.jsonl` into a list[dict], instantiate
   `FacapDataset(category="dress", split="train")` for image rendering
   and target-caption lookup. Print N queries + sanity check on first
   row keys.
4. **Quick stats** — N queries, R@K from `metrics.json`, rank
   distribution histogram (matplotlib). Confirms we're looking at the
   right run.
5. **Caption length stats** — side-by-side histograms: generated
   caption char length vs target caption char length. Single quick
   read on whether the VLM is over- or under-describing.
6. **Mentor gallery cell** (HEADLINE) — function
   `render_query(row) -> None` that produces a clean side-by-side per
   query:
   - left: reference image + modification text
   - middle: generated_caption (the VLM output)
   - right: target image + target_caption (ground truth) + rank +
     top-1 retrieved id (and optionally its caption)
   Driver loop renders 10 random queries (seed-fixed) and 5 hand-picked
   ones (rank=1 success, top-10, mid, deep miss, missing-from-DB).
7. **Stratified gallery** — same renderer, but bucket queries by rank
   (rank=1, 2-10, 11-50, 51-200, 201+, missing) and render 3 random
   examples per bucket with the seed fixed for reproducibility. The
   "what does a rank=1 caption look like vs a rank=500 caption" view.
8. **Caption-vs-target programmatic metrics** — for every row,
   compute:
   - `len_ratio = len(generated) / len(target)`
   - `token_overlap = |gen_tokens ∩ target_tokens| / |target_tokens|`
     (lowercased whitespace tokens, stop-words optional)
   - optional `bleu_score` if `nltk` is importable; skip cleanly if not
   Plot two scatters: rank vs token_overlap, rank vs len_ratio. Print
   Spearman correlation. Answers "do good captions correlate with
   good retrieval?" empirically.
9. **Failure annotation scaffold** (LAST, low-priority) — single
   helper:
   ```python
   def annotate(query_id: str, category: str, notes: str = ""): ...
   ```
   that appends `{query_id, category, notes, ts}` to
   `runs/baseline_v1_speechqwen2vl/annotations.jsonl`. Plus a markdown
   block listing the rubric categories (`caption_wrong`,
   `embedding_mismatch`, `dataset_ambiguity`, `visual_nuance_lost`)
   and a commented example call. Reader can opt in by uncommenting.

  the inline code.

## Verification

1. **End-to-end execute**:
   ```bash
   conda activate fashion_retrieval
   jupyter nbconvert --to notebook --execute --inplace \
       notebooks/caption_analysis.ipynb
   ```
   Should complete without errors against the existing
   `runs/baseline_v1_speechqwen2vl/` run.
2. **Visual spot-check**: open `caption_analysis.ipynb` in JupyterLab,
   confirm the mentor gallery cell renders images + captions cleanly
   (not just text dumps).
3. **Stratified cell**: every rank bucket has at least 1 example
   rendered (assert on bucket sizes).
4. **Metric cell**: rank-vs-overlap scatter actually plots; Spearman
   correlation is a real number, not NaN.
5. **Re-pointing**: change `RUN_DIR` to
   `runs/baseline_v1_speechqwen2vl_marqo-fashionclip` in cell 2, re-run.
   Notebook should produce the analogous analysis for that encoder run
   without code changes (since per-encoder runs reuse the same
   generated_captions, the caption stats are identical but the rank
   stratification differs — this is a feature, not a bug, and it
   demonstrates the encoder-vs-caption attribution argument).
6. **Mentor-share dry run**: `jupyter nbconvert --to html
   notebooks/caption_analysis.ipynb` produces a self-contained HTML
   that renders all images and captions inline.
