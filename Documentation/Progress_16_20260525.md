## Summary

One Gradio app, two switchable tabs:

- **✨ v2 (new)** — the redesigned **A + B = C** equation view. Reference
  image ➕ modification 🟰 the model's matching garment. Mode selector for
  P1 / P2 / P3 / P4 / **🎬 Demo** (compares all four). Default tab.
- **📦 v1 (stable)** — the original stacked four-pipeline demo, frozen as a
  known-good fallback while v2 iterates.

Both tabs read cached results from `runs/demo/preset_cache.json`; **P4 live
recording** runs the Plan-15 audio two-tower in-process when `LIVE_AUDIO=1`.

This doc is the **operations manual** — how to start the server, env vars,
host requirements, access from a laptop, and known caveats.

## 1. Launching

```bash
bash scripts/run_demo.sh
```

Runs `python -m src.demo.app`, builds both tabs, serves on
`http://0.0.0.0:7860`. Stop with `Ctrl-C`.

### Environment variables

| Var | Default | Effect |
|---|---|---|
| `LIVE_AUDIO` | `0` | `1` enables v1's live-record row **and** v2's P4 live retrieval. Loads the Plan-15 audio two-tower (~9 B params) into GPU memory at startup (~10–30 s). Needs one ≥24 GB GPU. |
| `LIVE_AUDIO_DEVICE` | `cuda:0` | Which device the audio model lives on. |
| `CUDA_VISIBLE_DEVICES` | (unset) | Restrict which physical GPUs the process sees. Recommended to pin to a single free GPU. |
| `GRADIO_SERVER_PORT` | `7860` | HTTP port. |
| `GRADIO_SHARE` | `0` | `1` opens a `*.gradio.live` public URL (72 h). **Don't** flip this without explicit need — it exposes the demo publicly. |
| `DEMO_STAGE` | `v0.1` | Legacy v1 staging (`v0.1` = cached, `v0.2` = +Whisper live, `v0.3` = +P1 live). Not used by v2. |
| `HF_HOME` / `TRANSFORMERS_CACHE` | `/ssd1/zhuoyuan/hf_cache` (auto when `LIVE_AUDIO=1`) | HuggingFace cache. The launch script pins these to a local SSD so the ~9 B base never lands on NAS home. |

### Typical launches

Preset-only, no GPU (any host, including a laptop):
```bash
bash scripts/run_demo.sh
```

With P4 live recording (need a GPU host with ≥24 GB free):
```bash
LIVE_AUDIO=1 CUDA_VISIBLE_DEVICES=0 bash scripts/run_demo.sh
```

Custom port:
```bash
GRADIO_SERVER_PORT=7861 bash scripts/run_demo.sh
```

### Required local data

Always:
- `runs/demo/preset_cache.json` — cached results for the 8 preset examples
  (P1 / P2 / text2t / audio2t). Without it, `app.py` raises at startup.
- `runs/demo/preset_thumbs/` — bundled Top-K thumbnails (so the demo can run
  on a laptop without the full gallery on disk).
- `runs/demo/preset_audio/preset_0?.wav` — TTS clips for v1's preset audio
  player and v2's voice-clip picker.

When `LIVE_AUDIO=1` (assumes a lab GPU host):
- `/ssd1/<user>/plan15_demo/ckpt_epoch17/` — audio two-tower checkpoint
  (`head_query.pt`, `head_target.pt`, `logit_scale.pt`, `shared_backbone/`).
- `/ssd1/<user>/plan15_demo/gallery_emb_epoch17.npy` (+ `.meta.json`) —
  target-tower gallery embeddings (~115 MB).
- `/ssd1/<user>/facap-images/` — the ~59 k full-resolution gallery; live mode
  retrieves arbitrary IDs from this directory.

Paths above are the current lab-host conventions; edit `src/demo/config.py`
(`AUDIO_2T_CKPT`, `AUDIO_2T_GALLERY`, `GALLERY_DIR`) to point elsewhere.

## 2. Accessing the demo from your laptop

The server binds to `0.0.0.0:7860`, but cross-host HTTP is normally firewalled.
Use an **SSH tunnel** from your local machine:

```bash
ssh -N -L 7860:localhost:7860 <demo-host>
```

`-N` = forward only, no shell. Leave it running, open `http://localhost:7860`
in your browser, `Ctrl-C` to close. Behind a jump server:

```bash
ssh -N -L 7860:localhost:7860 -J <jump-host> <demo-host>
```

## 3. What's in each tab

### ✨ v2 (new) — default

The A + B = C demo. Click an example on the right; the reference image
(+ modification, for Demo mode) loads on the left; the model's matching
garment appears beside Ground Truth.

Top:
- **Instruction header** explaining the flow.
- **Mode selector**: P1 / P2 / P3 / P4 / **🎬 Demo**. Demo is the default.
- **❓ Pipeline explainer** accordion — engineer-grade step-by-step per
  pipeline (offline embed index → online encode → cosine NN search), with
  the R@10 scores.

Equation row:
- **🖼 Reference Image** — `gr.Image(sources=["upload"])`. Click the box (or
  the ➕ dashed button below) to upload your own; clicking an example fills
  it from the right panel.
- `+` glyph (CSS-centered to match the image's vertical mid-line).
- **📝/🔊 Modification** — modality-aware: text models (P1/P2/P3) show the
  typed-text box; the audio model (P4) shows the spoken-clip player
  (recordable in P4); 🎬 Demo shows both.
- **Result column** — single mode shows the model's Top-1; Demo shows a
  four-thumbnail "ALL FOUR top-1" comparison strip.
- **✅ Ground truth** beside the result for direct comparison.

Below the equation:
- **▶ Run** + **✖ Clear** buttons.
- **K slider** (always visible).
- **Single-mode Top-K** gallery (P1–P4).
- **🎬 Demo Top-K** — four labeled galleries (P1 / P2 / P3 / P4) stacked,
  each showing that pipeline's full ranked list under the K slider.

Right panel:
- **🖼 Garments** thumbnail gallery (always).
- **📝 Text prompts** — 8 buttons, each labeled with the example's
  modification snippet (visible in text modes + Demo).
- **🔊 Voice clips** — 8 buttons (visible in P4 + Demo).

Clicking any picker (garment, text prompt, or voice clip) loads the same
matched example — no mismatch possible.

**P4 live recording**: when `LIVE_AUDIO=1`, in P4 mode the audio box is
recordable (mic + upload). ▶ Run encodes (reference image + the in-box clip)
through the audio two-tower live and returns Top-1 + Top-K + GT.

### 📦 v1 (stable)

The original stacked demo (`build_ui` in `src/demo/app.py`):

- 8 preset thumbnails up top, click to pick.
- Active query area (image, modification text, mic, transcript, ground truth).
- Top-K slider + **🔍 Run ALL pipelines** button.
- Stacked sections per pipeline: P1 caption → P2 direct → P3 text two-tower →
  P4 audio two-tower, each with its own Run button and Top-K gallery.
- At the bottom (when `LIVE_AUDIO=1`): a **🎙 Live Audio Retrieval** row that
  records your voice and retrieves with the audio two-tower.

Frozen — kept identical to commit `7bf80ce` so the demo always has a
known-good fallback while v2 iterates.

## 4. Known limitations — deploy it yourself

Custom inputs in v2 are deliberately **UI-only** in this build:

- **Uploading a custom reference image** (drag, click the ➕ button, or click
  the empty box) is accepted and displayed, but it **does not retrieve**
  because no inference server is deployed for the image-text pipelines. The
  status surfaces this honestly and points at the in-app warning accordion.
- **Typing custom text** in the modification box: same — UI accepts it, but
  no retrieval.
- **Recording custom audio** works **for P4 only** (when `LIVE_AUDIO=1`),
  because the audio two-tower is loaded into the same process.

To wire the rest: clone the repo, deploy P1 / P2 / P3 behind an inference
server, and replace the bodies of `on_custom_image`, `on_upload`, and
`on_custom_text` in `build_ui_v2` to call it. The entry points and output
plumbing are already in place.

## 5. Troubleshooting

**Port already in use** — another demo process is bound. Find and kill:
```bash
pgrep -af "python -m src.demo.app"
kill -9 <pid>
```

**Live audio's first run is slow (~20 s)** — CUDA warmup on the first forward
pass after the audio model loads. Subsequent runs are ~1 s.

**Cached results missing / `FileNotFoundError: preset cache not found`** —
`runs/demo/preset_cache.json` was not populated on this host. Regenerate:
```bash
python -m src.demo.precompute_presets       # P1 + P2 entries
python -m src.demo.precompute_two_tower     # text2t + audio2t entries
```
Both need GPU + the relevant checkpoints; defaults match the lab-host paths
in `precompute_two_tower.py`.

**`TRANSFORMERS_CACHE is deprecated` warning** — harmless. The launch script
sets both `HF_HOME` and `TRANSFORMERS_CACHE` for cross-version compatibility.

**Image-box click doesn't open the file picker** — there is a small JS shim
in `build_ui_v2` (document-level capture-phase delegation) that routes clicks
on `#v2-ref-img` to the UploadButton's hidden `<input type="file">`. If a
future gradio upgrade changes its DOM, look for `__v2_img_click_wired` in the
`demo.load(... js=...)` block.

**Gradio share URL not opening** — `GRADIO_SHARE=1` must be set in the env
*before* launch; flipping it afterwards has no effect. Re-launch.

## 6. File map

| Path | Role |
|---|---|
| `scripts/run_demo.sh` | Launch script: env-var pinning, HF cache, etc. |
| `src/demo/app.py` | Both tabs. `build_ui()` = v1; `build_ui_v2()` = v2; `main()` joins them with `gr.TabbedInterface` and launches. |
| `src/demo/config.py` | Preset paths, cache dirs, `LIVE_AUDIO` flags, audio checkpoint / gallery locations. |
| `src/demo/pipelines/two_tower.py` | Audio two-tower loader + `run_two_tower_inference` (used by both v1's live row and v2's P4 live path). |
| `src/demo/precompute_presets.py` | Offline pass that populates the P1 / P2 entries in `preset_cache.json` (caption + direct embedding). |
| `src/demo/precompute_two_tower.py` | Offline pass that populates the text2t + audio2t entries. |
| `runs/demo/preset_cache.json` | What the demo actually reads at request time. |
| `runs/demo/preset_thumbs/` | Bundled Top-K thumbnails (so v1 can run on a laptop). |
| `runs/demo/preset_audio/` | TTS clips per preset. |
